# Provve Backend

## Модули системы

- **notification**: Управление уведомлениями и e-mail рассылками.
- **validation**: Прием решений, расчёт итоговых результатов.
- **payment**: Интеграция с платежным шлюзом, обработка заявок на оплату.
- **skill**: Опериррование навыками, экаменами и голосованиями (отдаёт команды другими модулями по результату голосования).
- **api**: Лицо системы. Обработка запросов верхнего уровня, разделение на подзадачи для отдельных модулей, без встроенной бизнес-логики.
    - **server**: Точка сборки. Этот модуль собирается и запускается.
- **account**: Оперирование аккаунтами, разграничение доступа.

## Архитектура

1. Вся система построена по структуре модульного монолита.
2. Каждый из модулей содержит всю логику своей части: от контроллеров до контекста домена. Это позволит развернуть
   модули как самостоятельные микросервисы в будущем, когда потребуется масштабировать одну из функциональностей.
